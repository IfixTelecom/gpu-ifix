## Gaps Summary

**Root cause:** A single confirmed gateway bug — `PRIMARY_VAST_MACHINE_ALLOWLIST` does not steer offer selection at runtime — blocks the phase from achieving its primary headline goal (end-to-end gateway provision of a 2×3090 to markReady). The code for allowlist-first search exists in both `types.go` (WithMachineAllowlist, L269) and `reconciler.go` (L784-796), but the runtime evidence from multiple force-up cycles unambiguously shows selection always broadens to the global-cheapest qualifying offer, bypassing the allowlist entirely.

This compounds with a second real-world condition: broken CDI is widespread on 2×3090 Vast hosts (confirmed: 55942 Michigan, 45778 Czech). With allowlist targeting broken, the gateway cannot be steered to the one validated host (43803 Estonia). Even hosts that don't fail CDI may be too geographically distant for the weight-download budget (76546 California ~50min cold-start).

**What IS proven:**
- STT-FIX: fully complete and promoted. `/v1/audio/transcriptions` returns 200 on the `:develop` image (GPU offload confirmed). Canonical MinIO key updated, GH Secrets updated, dev-stack env updated. Source committed to develop (commits 5635502 + 999e290).
- GW-2GPU mechanics: the gateway correctly searches for 2×3090 offers through the cuda/driver gates (7 qualifying offers), creates Vast instances, maps 4 service ports, detects CDI failures and auto-destroys. Cost guard holds (price cap 0.60, MONTHLY_PRIMARY_BUDGET_BRL guard active). The SEED-002 shape input (2×RTX 3090 48GB) is validated.
- LADDER: operator runbook complete with per-shape presets, transition procedure, Shape C deferred, allowlist seeded.

**What is MISSING for full GW-2GPU:**
1. Fix the allowlist-first preference pass so it actually restricts the search to preferred machine_ids before broadening.
2. Re-run the Plan 03 live UAT targeting machine 43803, achieving markReady + LLM/STT/TTS/DCGM 200 + nvidia-smi 2-GPU split.

The bug is documented in `.planning/todos/pending/260522-allowlist-steering-bug.md` with the fix prescription and re-test requirements.
