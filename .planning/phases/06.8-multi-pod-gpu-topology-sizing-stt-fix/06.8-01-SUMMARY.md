# Phase 06.8 Plan 01: STT fix — whisper HF-hub-cache layout + HF_HUB_CACHE Summary

Prepares the STT fix (RESEARCH fix iii) at script/image level: `upload-weights.sh` now emits the whisper tarball in HuggingFace-hub-cache layout (with README.md model card + per-file integrity guards + a candidate-key path), and the speaches supervisord child gets `HF_HUB_CACHE=/weights/whisper` so its `get_model_card_data_or_raise()` gate resolves the pre-shaped cache. Changes are committed to the worktree branch ONLY — NOT merged to develop (the develop commit is gated on Plan 02's live validation per CLAUDE.md no-blind-commit-to-dev).

## What Was Built

### Task 1 — Whisper tarball in HF-hub-cache layout + integrity + candidate key (`upload-weights.sh`)
Commit `c6d7c3b`.
- New `hf_download_checked()` helper: downloads via existing `hf_download`, then asserts the file is present + non-empty (`[ -s ]`), and logs its sha256 + byte size. A truncated/corrupt/0-byte download ABORTS the script (explicit guard on top of `set -e`), so a partial file can never enter the tarball (T-06.8-01 mitigation).
- Whisper block rewritten: resolves the REAL HF main commit hash via `curl … api/models/Systran/faster-whisper-large-v3 | jq -r .sha`, builds a stage dir with `models--Systran--faster-whisper-large-v3/refs/main` (content = hash, no trailing newline) + `snapshots/<hash>/`, downloads `model.bin config.json tokenizer.json vocabulary.json preprocessor_config.json README.md` (README.md is now MANDATORY — previously never fetched; without it speaches 500s MODEL_CARD_DOESNT_EXISTS), as REAL file copies (not symlinks), then `tar -C "${WHISPER_STAGE}"` so the tarball top-level entry is `models--Systran--faster-whisper-large-v3/` (NOT flat).
- Candidate-key support: a header comment block + an `IS_CANDIDATE_RUN` flag (set when `WEIGHTS_VERSION != v1.0.0`). On a candidate run, ONLY the whisper artifact is produced under the candidate key (`whisper-large-v3/v1.0.0-hf-cache-candidate/model.tar.gz`); Qwen + BGE-M3 blocks are SKIPPED so canonical `v1.0.0` keys are untouched. Final-instructions output is candidate-aware.

### Task 2 — `HF_HUB_CACHE` on the speaches supervisord child (`supervisord.conf`)
Commit `454b3f9`.
- `[program:speaches]` gains `environment=HF_HUB_CACHE="/weights/whisper"` (absolute path, matching `onstart.go`'s `tar -xzf … -C /weights/whisper` exactly — Pitfall 4), placed directly under `command=`.
- `directory=/opt/speaches-data` preserved (UAT 11). No `WHISPER__INFERENCE_DEVICE` (left absent → speaches auto→CUDA on the 2×3090 / 5090 headroom). No model-dir vars introduced.
- Comment block updated: the HF cache is now PRE-POPULATED by the pre-shaped tarball at `/weights/whisper`, superseding the old "populated via POST /v1/models/{id}" runtime-fetch note.

## Whisper commit hash (for Plan 02's A1 check)
The hash is NOT hardcoded — `upload-weights.sh` resolves it at run time from `https://huggingface.co/api/models/Systran/faster-whisper-large-v3` (`.sha`) and prints it in the final-instructions block ("Whisper repo commit hash baked into the cache layout: <hash>"). It is also recoverable from the tarball at `models--Systran--faster-whisper-large-v3/refs/main`. Plan 02 reads it from the candidate run's output (or `refs/main`) to perform the A1 arbitrary-hash check in-pod.
