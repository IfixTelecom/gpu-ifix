## Summary

Esta fase tem três frentes, todas em código/config + UMA validação obrigatória em pod vivo (o STT fix). Nenhuma é greenfield — tudo encosta em código já existente e bem comentado.

**STT (a incógnita de maior valor):** o root cause está 100% traçado no source pinado do speaches `v0.9.0-rc.3`. O router de transcrição (`src/speaches/routers/stt.py`) chama `get_model_card_data_or_raise(model)` ANTES de qualquer load do modelo. Essa função (`routers/utils.py` → `hf_utils.get_model_repo_path`) itera o diretório `HF_HUB_CACHE` procurando uma pasta `models--Systran--faster-whisper-large-v3/` e, dentro dela, `snapshots/<hash>/README.md` (model card). O `HF_HUB_CACHE` default é `/root/.cache/huggingface/hub` (que não existe no pod → `CacheNotFound`), e o tarball atual extrai os arquivos FLAT em `/weights/whisper` (invisível a essa busca; nem inclui README.md). Logo: o fix correto e mínimo é **(iii) regenerar o tarball do whisper em layout HF-hub-cache (incluindo README.md) + setar `HF_HUB_CACHE=/weights/whisper` (ou outro caminho) no env do `[program:speaches]`** — descrito exatamente abaixo. As opções (i) e (ii) têm armadilhas que documento.

**Gateway 2×3090:** os dois knobs (`PRIMARY_NUM_GPUS`, `PRIMARY_VAST_MACHINE_ALLOWLIST`) estão landados e o caminho `provisionLifecycle → buildCreateRequest → CreateInstance → waitForReadyOrDestroy` é agnóstico de número de GPUs (porta-publish via env-keys, health-poll via host ports). Os ÚNICOS gaps reais são de **configuração de runtime**, não de código: `PRIMARY_GPU_NAME` default "RTX 5090" precisa virar "RTX 3090" e `PRIMARY_VAST_PRICE_CAP_DPH` default 2.20 cobre ~$0.40 (não bloqueia). Recomendo um live-UAT via `gatewayctl primary force-up` espelhando o spike manual.

**Fallback ladder:** dado que o gateway é single-primary e o reconciler já tem o broaden-fallback (allowlist→qualified search) + cooldown, uma lista config-driven de topologias seria sobre-engenharia agora. Recomendo um **runbook operador + env-presets por shape** (2×3090 → 5090 → 1pod-LLM+tiny-GPU), com a transição entre shapes feita por troca de env + `force-up`.

**Primary recommendation:** STT fix (iii) tarball-em-HF-cache-layout + `HF_HUB_CACHE`; gateway 2×3090 via env (`PRIMARY_NUM_GPUS=2`, `PRIMARY_GPU_NAME="RTX 3090"`, `PRIMARY_VAST_MACHINE_ALLOWLIST=43803`) + live-UAT `force-up`; ladder como runbook+presets.

## Architecture Patterns

### System Architecture Diagram (STT resolution chain — onde o bug vive)

```
POST /v1/audio/transcriptions (model="Systran/faster-whisper-large-v3")
        │
        ▼
routers/stt.py: transcription handler
        │  (1) get_model_card_data_or_raise(model)   ◄── GATE (roda ANTES do load)
        ▼
routers/utils.py: get_model_card_data_or_raise
        │  → hf_utils.get_model_repo_path(model_id)
        ▼
hf_utils.get_model_repo_path:
        cache_dir = HF_HUB_CACHE  (default /root/.cache/huggingface/hub)
        │
        ├─ se cache_dir NÃO existe → raise CacheNotFound  ◄── O ERRO ATUAL (500)
        │
        └─ itera cache_dir.iterdir() procurando "models--Systran--faster-whisper-large-v3"
                 └─ se não acha → 404 "not installed locally"
                 └─ se acha → _scan_cached_repo() lê snapshots/<hash>/README.md (model card)
                                  └─ sem README.md → 500 MODEL_CARD_DOESNT_EXISTS
        │  (2) só DEPOIS de passar o gate:
        ▼
executors/whisper.py: WhisperModelManager._load_fn(model_id)
        → WhisperModel(model_id, device=..., ...)   (model_id = repo-id string)
        → faster-whisper resolve via HF cache (snapshot já presente)
```

O fix tem que satisfazer (1) o GATE — criar o layout `models--…/snapshots/<hash>/` COM README.md dentro de `HF_HUB_CACHE`.

### Provisioning flow (gateway — agnóstico de GPU count, já pronto)

```
gatewayctl primary force-up → PUBLISH gw:primary:events {force_up_request}
        ▼
reconciler.handleForceUpRequest → spawnProvisioning
        ▼
provisionLifecycle:
   filter = DefaultSearchFilter(priceCap, hostID, PrimaryGPUName, PrimaryNumGPUs, blocklist...)
   if allowlist set: SearchOffers(WithMachineAllowlist(filter)) → broaden if empty
   pick cheapest → buildCreateRequest(offer)
        env: "-p 8000:8000":"1", "-p 8001:8001":"1", "-p 8003:8003":"1", "-p 9400:9400":"1"
             + MinIO 4 + 3 weight key/sha pairs ; Disk:50 ; Runtype:"args" ; Entrypoint:"/bin/bash"
   CreateInstance → waitForReadyOrDestroy:
        poll GetInstance → quando running, build 4 URLs via inst.Ports["<port>/tcp"][0].HostPort
        4-endpoint HealthCheck (LLM /v1/models, STT /health, TTS /health, DCGM /metrics) → markReady
```

### Pattern: env-knob espelhando o blocklist (estabelecido)
**What:** novos knobs (`PRIMARY_NUM_GPUS`, `PRIMARY_VAST_MACHINE_ALLOWLIST`) seguem o padrão `parseInt64CSV`/`atoiOr` no config + aplicação em `DefaultSearchFilter`/`With*` helper.
**When to use:** já feito; só precisa SETAR os valores de runtime para 2×3090.
**Source:** `gateway/internal/config/config.go:368-372`, `gateway/internal/emerg/vast/types.go:229-283`.

### Anti-Patterns to Avoid
- **Blind-edit do STT no pod:** CLAUDE.md proíbe edits especulativos do debugger. O fix é image/script-level mas a VALIDAÇÃO é obrigatória em pod vivo antes de commitar para dev.
- **Pinar host na allowlist como hard filter:** o reconciler já faz preference+broaden de propósito; transformar em hard filter quebra provisioning quando o host está busy (Vast é spot).
- **Mexer no pin de `faster-whisper==1.1.1`:** está sandwiched entre requisitos (UAT 17). Não tocar.

## STT Fix — fix correto + arquivos/envs exatos

### Root cause (VERIFIED via source pinado)
- **Request:** `pod/smoke/smoke.py:262` envia `{"model": "Systran/faster-whisper-large-v3"}`. [VERIFIED: codebase]
- **Gate:** `routers/stt.py` (handlers de transcription/translation) chamam `get_model_card_data_or_raise(model)` ANTES do load. [CITED: github.com/speaches-ai/speaches/blob/v0.9.0-rc.3/src/speaches/routers/stt.py]
- **Resolução:** `routers/utils.py:get_model_card_data_or_raise` → `hf_utils.get_model_repo_path(model_id)` → itera `HF_HUB_CACHE` por `models--Systran--faster-whisper-large-v3/`; depois `_scan_cached_repo` + `get_model_card_data_from_cached_repo_info` exigem `snapshots/<hash>/README.md`. [CITED: .../v0.9.0-rc.3/src/speaches/hf_utils.py L137-167, L80-95]
- **Default cache:** `HF_HUB_CACHE` (do `huggingface_hub.constants`) = `~/.cache/huggingface/hub` = `/root/.cache/huggingface/hub`. Não existe no pod → `raise huggingface_hub.CacheNotFound`. [CITED: hf_utils.py L143-148]
- **Load:** só após o gate, `executors/whisper.py:_load_fn` faz `WhisperModel(model_id, ...)` — passa a **repo-id string**, NÃO um path. [CITED: .../v0.9.0-rc.3/src/speaches/executors/whisper.py L133-141]

### Por que (ii) é inviável e (i) é frágil
- **(ii) path local:** `_load_fn` não aceita path (passa `model_id`); e mesmo que aceitasse, o GATE `get_model_repo_path` roda antes e exige o layout de cache. `WHISPER__MODEL_DIR`/`SPEACHES__MODEL_DIR` não existem no `WhisperConfig` (UAT 15 já confirmou — `supervisord.conf` L52-54). **Descartar.**
- **(i) reshape no onstart:** funciona em teoria, mas exige criar `models--Systran--faster-whisper-large-v3/snapshots/<hash>/` com `model.bin config.json tokenizer.json preprocessor_config.json README.md` + `refs/main` apontando pro `<hash>`, em bash, no caminho crítico de boot. Mais código frágil. **Não recomendado.**

### Fix recomendado: (iii) tarball em HF-hub-cache layout + `HF_HUB_CACHE`
**Mudança 1 — `pod/scripts/upload-weights.sh` (whisper block, L80-91):** gerar o tarball já com a estrutura de cache em vez de flat. A estrutura mínima que satisfaz speaches:
```
models--Systran--faster-whisper-large-v3/
├── refs/
│   └── main                      # conteúdo: <commit_hash> (qualquer hash estável; pode ser o real do HF)
└── snapshots/
    └── <commit_hash>/
        ├── model.bin
        ├── config.json
        ├── tokenizer.json
        ├── preprocessor_config.json
        ├── vocabulary.json        # (se existir)
        └── README.md              # OBRIGATÓRIO — model card; speaches falha 500 sem ele
```
- Os arquivos em `snapshots/<hash>/` podem ser **cópias reais** (não precisam ser symlinks para `blobs/` — `_scan_cached_repo` lida com ambos; cópias evitam o problema de symlink quebrar no tar/untar).
- **Baixar também o `README.md`** de `https://huggingface.co/Systran/faster-whisper-large-v3/resolve/main/README.md` (o script atual NÃO baixa). Sem ele → 500 `MODEL_CARD_DOESNT_EXISTS`.
- `tar -C <stage>` onde `<stage>` contém a pasta `models--Systran--faster-whisper-large-v3/` no topo (não o conteúdo flat). [CITED: huggingface_hub cache docs — refs/blobs/snapshots layout]

**Mudança 2 — `pod/primary/supervisord.conf` `[program:speaches]`:** adicionar `HF_HUB_CACHE` apontando para onde o tarball é extraído. O onstart já extrai whisper para `/weights/whisper` (`onstart.go:159`). Então:
```ini
[program:speaches]
environment=HF_HUB_CACHE="/weights/whisper"
```
(o diretório `/weights/whisper` passará a conter `models--Systran--faster-whisper-large-v3/...` após o untar). NB: `[program:chatterbox]` já usa `environment=`; speaches hoje não tem env — adicionar a linha. Isto é IMAGE-LEVEL (rebuild da imagem).

**Mudança 3 (consequência):** o `WHISPER_SHA` muda (tarball novo) → atualizar `PRIMARY_WHISPER_WEIGHTS_SHA256` (env/GH Secret) com o novo hash que `upload-weights.sh` imprime.

### O que o executor DEVE validar AO VIVO no pod (não assumir)
1. **`HF_HUB_CACHE` honrado:** após boot, `ls /weights/whisper/models--Systran--faster-whisper-large-v3/snapshots/*/` mostra os 4-5 arquivos + README.md.
2. **Gate passa:** `curl localhost:8001/v1/models` lista o whisper como local model (`list_local_models` itera o cache).
3. **Transcrição real:** POST `/v1/audio/transcriptions` com `model=Systran/faster-whisper-large-v3` retorna 200 + texto (não 404 "not installed", não 500 card-missing, não `CacheNotFound`).
4. **GPU offload:** confirmar via `dcgm`/`nvidia-smi` que whisper carrega na GPU (não CPU) — em 2×3090 há headroom (48GB).
5. **silero v5 assets:** confirmar que os ONNX vendored (Dockerfile L177-182) continuam presentes — o fix não os toca, mas a transcrição depende deles (UAT 15).
6. **Validar se `<commit_hash>` arbitrário é aceito** ou se speaches/HF exige o hash real do repo — se `refs/main` + a pasta snapshot baterem, `get_model_card_data_from_cached_repo_info` aceita (`revisions[0]` quando há 1 revisão). Confirmar empiricamente.

## Common Pitfalls

### Pitfall 1: README.md ausente no tarball whisper
**What goes wrong:** mesmo com o layout `models--…/snapshots/<hash>/` correto, falta o README.md → speaches 500 `MODEL_CARD_DOESNT_EXISTS`.
**Why:** `get_model_card_data_from_cached_repo_info` exige o README.md (model card) — não é opcional.
**Avoid:** baixar `README.md` no `upload-weights.sh` (o script atual NÃO baixa).
**Warning sign:** erro muda de `CacheNotFound` para `MODEL_CARD_DOESNT_EXISTS` após setar `HF_HUB_CACHE` — significa que o layout está certo mas falta o card.

### Pitfall 2: symlinks no tarball
**What goes wrong:** se o tarball usar symlinks `snapshots→blobs` (layout HF real) e o tar/untar quebrar os links → arquivos vazios.
**Avoid:** usar cópias reais em `snapshots/<hash>/` (speaches lê arquivos diretamente; não exige a indireção blobs).

### Pitfall 3: gate `cuda_max_good`/`driver_vers` zera offers de 2×3090
**What goes wrong:** análogo ao UAT 18 (5090 + `cuda_max_good gte 12.6` retornava zero). A busca filtrada pode retornar 0 offers mesmo havendo 2×3090 baratos.
**Avoid:** VALIDAR ao vivo que `DefaultSearchFilter(..., "RTX 3090", 2, ...)` retorna offers; se zerar, investigar `cuda_max_good`/`driver_vers` (o spike usou create MANUAL, que NÃO passa pelos gates de busca).
**Warning sign:** `provisionLifecycle` fecha com `no_offers_below_cap` apesar de inventory existir.

### Pitfall 4: `HF_HUB_CACHE` no env errado
**What goes wrong:** setar `HF_HUB_CACHE` global mas o onstart extrai whisper para outro path, ou o speaches roda com `directory=/opt/speaches-data` e um path relativo.
**Avoid:** `HF_HUB_CACHE` deve ser **absoluto** (`/weights/whisper`) e bater EXATAMENTE com o `tar -xzf ... -C /weights/whisper` do onstart (`onstart.go:159`). Confirmar in-pod.
