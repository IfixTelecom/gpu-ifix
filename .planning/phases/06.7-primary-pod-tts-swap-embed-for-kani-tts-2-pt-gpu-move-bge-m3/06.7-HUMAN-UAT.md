## Scenario S2 — Zero-shot voice-clone round-trip (24kHz cloned voice)

**Goal:** Upload reference WAV → `voice_id` → `/v1/audio/speech` returns 24kHz audio in the cloned voice.

```bash
# upload reference WAV
curl -sS -H "Authorization: Bearer $KEY" \
  -F "file=@reference.wav" -F "label=uat-voice" \
  $GW/v1/audio/voices
# -> {"voice_id":"<uuid>","label":"uat-voice"}   (export VID=<uuid>)

# synth in that voice
curl -sS -H "Authorization: Bearer $KEY" -H "Content-Type: application/json" \
  -d "{\"model\":\"chatterbox\",\"input\":\"Olá, seu boleto venceu ontem.\",\"voice\":\"$VID\",\"response_format\":\"wav\"}" \
  $GW/v1/audio/speech -o s2_cloned.wav
file s2_cloned.wav    # expect: RIFF ... WAVE audio ... 24000 Hz, 16 bit, mono
# verify the S3 object exists
mc ls ifix/ai-gateway/voices/$VID.wav
```

**Expected:** `voice_id` returned; `s2_cloned.wav` is a 24000 Hz WAV; the voice audibly matches the reference (operator listens). MinIO has `voices/$VID.wav`.

- **Record:** voice_id = __________ · audio sample-rate = ____ Hz · clone match: yes/no.

## Scenario S3 — VOICE SURVIVES POD REPLACEMENT (S3 WAV refetch, NO `.pt`) [REVIEWS #1]

**Goal:** With the `voice_id` from S2, destroy + re-provision the pod (fresh ephemeral storage), then synth with the SAME `voice_id` — the cloned voice still works because the pod refetches the reference WAV from S3 as `audio_prompt_path`. Confirm logs show a **cache miss → S3 refetch → audio_prompt_path** (no embedding regen, no `.pt`).

```bash
# force the pod down + back up (fresh pod, wiped local voice cache)
docker exec ai-gateway-dev_gateway /gatewayctl primary force-down
# wait drain complete + Vast instance destroyed, then:
docker exec ai-gateway-dev_gateway /gatewayctl primary force-up --reason "06.7_uat_s3_replace"
# wait Ready (fresh pod)

# synth with the SAME voice_id from S2
curl -sS -H "Authorization: Bearer $KEY" -H "Content-Type: application/json" \
  -d "{\"model\":\"chatterbox\",\"input\":\"O pagamento foi confirmado, obrigado.\",\"voice\":\"$VID\",\"response_format\":\"wav\"}" \
  $GW/v1/audio/speech -o s3_survived.wav
file s3_survived.wav    # expect 24000 Hz WAV, same cloned voice

# confirm the pod log shows cache-miss -> S3 refetch (NO .pt, NO embedding regen)
ssh -p <pod_ssh_port> root@<pod_ip> 'tail -n 200 /var/log/supervisor/chatterbox*.log | grep -iE "cache miss|fetch.*s3|audio_prompt_path|voices/"'
```

**Expected:** `s3_survived.wav` is the same cloned voice (operator listens) on a brand-new pod. Pod log shows a cache MISS for `$VID` then an S3 fetch of `voices/$VID.wav` passed as `audio_prompt_path`. **No `.pt` file referenced, no embedding regeneration** — the durable artifact is the S3 WAV alone.
