**OpenAI `/v1/audio/speech` (tier-0 pod Chatterbox -> tier-1 pure-Go mu-law->WAV-16kHz Piper adapter) plus tenant-isolated zero-shot `/v1/audio/voices` CRUD on MinIO, wired through the role-based breaker dispatcher.**

## Accomplishments
- `NewTTSProxy`: tier-0 JSON->binary reverse proxy for the pod Chatterbox server (auth-strip via BuildDirector, OpenAI-shaped 502 via ErrorHandler("tts"), 60s ResponseHeaderTimeout, buffered for the single WAV body).
- `NewPiperTTSAdapter`: the GATE-3 Option A tier-1 fallback implemented verbatim — JSON `{input,voice}` -> Piper form `POST /tts {text,voice}`; raw mu-law 8kHz decoded via a pure-Go 256-entry G.711 LUT, upsampled 2x, wrapped in a 16kHz/16-bit/mono RIFF header (no ffmpeg); `response_format` wav (default) + pcm, unsupported (mp3/opus/flac/aac) -> clean OpenAI 400.
- `/v1/audio/voices` CRUD: first Go S3 consumer (minio-go v7), WAV-only persistence (no speaker-embedding file — Chatterbox zero-shot), tenant isolation via `auth.MustFromContext`, UUID-derived S3 keys (path-traversal safe), delete cleanup (S3 then row, 502-keep-row on S3 failure), idempotent upload (upsert-on-(tenant,label)).
- 5 enumerated TTS config env vars + the tts dispatcher mounted with the same tier-0->tier-1 breaker fallback wrap as chat/audio.
- All 9 owned Wave 0 stubs unskipped + asserting real behavior; +1 fallback integration test (tier-0 fail -> tier-1 Piper adapter converts to WAV 16kHz).

## Decisions Made
- minio-go v7 chosen over aws-sdk-go-v2 (lighter, MinIO-native; both sanctioned by threat T-06.7-SC).
- Idempotency = upsert-on-(tenant_id, label) — the schema has no Idempotency-Key store; an Idempotency-Key header is reserved for a future per-request dedup store and documented in the handler.
- 16kHz conversion target met by 2x sample duplication of Piper's 8kHz mu-law decode, keeping the adapter dependency-free (no DSP resampler).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] CreateVoice row id must equal the s3_key UUID**
- **Found during:** Task 2 (voices CRUD)
- **Issue:** The migration 0025 `id` defaults to `gen_random_uuid()` (DB-generated), but the handler derives the S3 key from a SEPARATE server-generated UUID. They would never match, so the pod (which fetches `<prefix>/<voice_id>.wav` keyed on the row id, Plan 05) would look for the wrong object — the voice would be unusable.
- **Fix:** Changed the `CreateVoice` query + generated code to accept a server-supplied `id`; the handler now passes the same UUID it derives the s3_key from, so `row.id == s3_key UUID`. The migration's DEFAULT is retained for any other insert path.
- **Files modified:** gateway/db/queries/voices.sql, gateway/internal/db/gen/voices.sql.go, gateway/internal/proxy/voices.go
- **Verification:** TestVoices_S3KeyFromUUIDNotInput asserts `s3_key` contains the row id; full proxy suite green.
- **Committed in:** ff959ca (Task 2 commit)

**2. [Rule 3 - Blocking] Empty UPSTREAM_TTS_URL at boot crashes NewTTSProxy**
- **Found during:** Task 3 (main.go wiring)
- **Issue:** UPSTREAM_TTS_URL is a placeholder the reconciler overrides at runtime (D-11), so it is empty at boot. NewTTSProxy rejects an empty/invalid URL with os.Exit(2).
- **Fix:** Fall back to a dead-localhost placeholder (`http://127.0.0.1:1`) when UPSTREAM_TTS_URL is unset; the breaker handles the unreachable tier-0 until the reconciler writes the live override (mirrors how the emergency-pod override works).
- **Files modified:** gateway/cmd/gateway/main.go
- **Verification:** `go build ./...` + cmd/gateway tests green.
- **Committed in:** f20f851 (Task 3 commit)

**3. [Rule 3 - Blocking] Added minio-go dependency + go.mod tidy**
- **Found during:** Task 2 (first Go S3 consumer)
- **Issue:** No S3 SDK was present in go.mod; the voices handlers need one. Threat T-06.7-SC explicitly sanctions adding a mainstream MinIO/AWS module (NOT an ambiguous/hallucinated package — the plan names it).
- **Fix:** `go get github.com/minio/minio-go/v7@v7.0.77` + `go mod tidy`, pinned in go.mod/go.sum.
- **Files modified:** go.mod, go.sum
- **Verification:** `go build ./...` green; voices tests use a hermetic fake VoiceObjectStore (no live MinIO).
- **Committed in:** ff959ca (Task 2 commit)
