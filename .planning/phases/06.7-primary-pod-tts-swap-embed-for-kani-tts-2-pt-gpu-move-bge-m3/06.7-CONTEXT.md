## ⚠ ROADMAP Premise Correction (validated 2026-05-19)

The ROADMAP 06.7 "Embed cross-model constraint" paragraph is **factually wrong** and MUST NOT be trusted by downstream agents.

**ROADMAP claims:** "bge-m3 (1024-dim) vector space ... The 24/7 CPU embed MUST use the SAME bge-m3 weights ... zero re-embed of the existing RAG corpus."

**Reality (validated against converseai-v4 source):** The existing converseai RAG corpus is embedded with **OpenAI `text-embedding-3-small` (1536-dim)**, called **directly** (not via the gateway). bge-m3 is NOT the corpus model. Evidence:
- `../converseai-v4/agents/src/rag/embeddings.py:5` — ingest uses OpenAI text-embedding-3-small
- `../converseai-v4/agents/src/rag/config.py:15` — `EMBEDDING_MODEL = "text-embedding-3-small"`, `EMBEDDING_DIMS = 1536`
- `../converseai-v4/agents/src/rag/retrieval.py:24-28` — query embed calls `AsyncOpenAI.embeddings.create`
- `../converseai-v4/packages/db/src/schema/rag.ts:72` — pgvector `embedding vector(1536)`, default model `text-embedding-3-small`

**Consequence:** The embed model for the 24/7 CPU service is an OPEN decision (researcher chooses best self-hostable model), and any cutover from OpenAI requires a corpus **re-embed** — deferred to Phase 8. Update ROADMAP 06.7 to reflect this.
</roadmap_correction>

<domain>
## Phase Boundary

Extend the peak-only primary pod (RTX 5090, supervisord PID 1 with 4 children from Phase 6.6: llama/speaches/infinity/dcgm) to add **GPU TTS via Kani-TTS-2-pt** (PT-BR, voice cloning) as a 5th supervisord child, and **relocate the embedding service off the pod onto a 24/7 CPU host** so RAG survives off-peak (pod sleeps 22h→8h + weekends).

Net result after this phase:
- Primary pod (peak-only GPU): LLM + STT + **TTS** (embed removed).
- Embed: runs **24/7 on CPU** (n8n-ia-vm), wired as a **static tier-0** gateway upstream.
- Gateway: new `/v1/audio/speech` proxy + `/v1/audio/voices` voice-clone CRUD + `tts` tier-0 role.
- voice-api Piper stays as TTS tier-1 fallback (degraded, default voice — no cloning).

**Engine LOCKED (research 2026-05-19):** Kani-TTS-2-pt (`nineninesix/kani-tts-2-pt`) — Apache 2.0, native PT-BR, zero-shot voice cloning via speaker embedding, ~3 GB VRAM, RTF 0.2.

**Not this phase:**
- Corpus re-embed + converseai OpenAI→gateway embed cutover + pgvector dim migration → **Phase 8** (INT-01).
- Switching the LLM/STT engines or models (Qwen3.6 + whisper-large-v3 stay as Phase 6.6).
- Gateway FSM/fallback-chain core redesign (orthogonal).
- Multimodal / vision.

<decisions>
## Implementation Decisions

### Embed — 24/7 CPU relocation

- **D-01:** **Host = n8n-ia-vm** (VMID 101, 10.10.10.20, 6c/16G/40G). This is also where converseai production will live. Operator will free resources or spin a dedicated VM if the chosen model's CPU/RAM footprint requires it — researcher MUST size the candidate models against this host.
- **D-02:** **Embed model is OPEN — researcher decides.** Evaluate best **self-hostable** embed model for PT-BR RAG by: PT-BR retrieval quality, CPU latency/throughput, RAM footprint. Candidates: bge-m3, multilingual-e5 (large/instruct), jina-embeddings-v3, **mistral-embed** (operator-requested — validate self-host vs API-only; decision is self-host, so flag if API-only). Recommend a model + serving engine (Infinity vs HF TEI vs sentence-transformers vs ONNX).
- **D-03:** **Wiring = static tier-0 upstream.** Embed becomes a permanent always-on tier-0 row (e.g. `UPSTREAM_EMBED_URL` → CPU host), and is **removed from the reconciler `OverrideTier0` map**. No FSM involvement. OpenAI embed remains tier-1 for **ephemeral** embed requests only (not persistent RAG).
- **D-04:** **Scope split — re-embed deferred to Phase 8.** 06.7 delivers the CPU embed service + model choice + static tier-0 wiring. The actual converseai corpus re-embed, the OpenAI→gateway embed cutover, and the `pgvector vector(1536)→new dim` migration are **Phase 8** (INT-01) work — flagged as now REQUIRING a dimension migration that the original Phase 8 plan did not account for.

### TTS — server + contract

- **D-05:** **TTS server = own thin FastAPI shim (~150 LoC)** loading Kani-TTS-2-pt via transformers, exposing OpenAI-compatible `/v1/audio/speech`. Runs as the **5th supervisord child** on the primary pod. We control the contract — no community wrapper dependency.
- **D-06:** **Contract = OpenAI `/v1/audio/speech`** (`model`, `input`, `voice`, `response_format`). Voice cloning is selected via the `voice`/`voice_id` field. Clients only swap `base_url` — consistent with the rest of the gateway.
- **D-07:** **Fallback = voice-api Piper (tier-1).** Off-peak (pod asleep) and on Kani failure, gateway falls back to voice-api Piper (CPU, 24/7). Piper does **not** voice-clone → degraded to a **default voice** in those windows (not an error). Researcher must confirm voice-api exposes a routable speech endpoint (adapter if not OpenAI-compat).

### Voice cloning — workflow + storage

- **D-08:** **Managed voices.** App uploads reference audio once via gateway `/v1/audio/voices` → reference WAV persisted to **MinIO (S3)**, returns a `voice_id` handle. Subsequent `/v1/audio/speech` requests pass `voice_id` (no re-upload, speaker embedding reusable).
- **D-09:** **Voice catalog = Postgres `ai_gateway` schema + S3.** Metadata row (`voice_id`, `tenant`, `label`, `s3_key`, `created_at`, …) in the shared DO Postgres `ai_gateway` schema (gateway standard); reference WAV in MinIO bucket.
- **D-10:** **Multi-tenant isolated.** Each `voice_id` belongs to a tenant (API key); apps see/use only their own voices — consistent with the gateway's per-app quota/cost model.

### TTS — gateway routing / tier-0

- **D-11:** **Add `'tts'` role to `OverrideTier0`.** Reconciler today overrides llm/stt/embed when the pod reaches Ready; becomes **llm/stt/tts** (embed leaves per D-03). The `tier0Override` map is role-agnostic (`gateway/internal/upstreams/loader.go:83` — "só o map precisa crescer") — add a `"tts"` key + a TTS probe. TTS cannot be a plain static upstream because the pod is peak-only and its IP changes per lifecycle — it needs the reconciler's dynamic override.
- **D-12:** **New TTS request pipeline (mirror existing patterns).** TTS needs its own: `/v1/audio/speech` route + reverse proxy (JSON→binary audio — opposite of the STT multipart→text path in `gateway/internal/proxy/audio.go`, cannot reuse), `tts` probe in `gateway/internal/upstreams/probe.go`, a TTS `RouteClass` in quota (`gateway/internal/quota/bucket.go` + `enforcer.go`), and the `/v1/audio/voices` CRUD handlers.
- **D-13:** **Pitfall #11 fix spans 3 roles.** Tech debt #9 (UAT 18): emerg `RestoreTier0` clears the tier-0 slot the primary wrote, and the primary reconciler doesn't auto-restore on Ready-tick. Adding `tts` to the map means this bug now affects TTS too. Researcher/planner design the fix so the primary reconciler re-calls `OverrideTier0` on each Ready-tick when the slot is empty, covering **llm/stt/tts** together.

### Claude's Discretion (planner/researcher derive)

- VRAM budget on the 5090 with the 4th GPU resident model (Kani ~3 GB on top of Qwen Q4_K_M + whisper-large-v3 GPU + KV cache). Researcher validates headroom; Phase 6.6 DCGM showed ~24.7 GB used / ~7.4 GB free at full load on the 32 GB 5090 — Kani should fit, but confirm.
- Output audio format/sample-rate defaults for `/v1/audio/speech` (Kani native vs OpenAI defaults).
- TTS latency SLO + Piper fallback trigger threshold — derive from existing probe/breaker config.
- supervisord program directives for the Kani child (autorestart, GPU device, model warmup).
- Exact env-var names for embed static upstream + TTS config.
