### Decision-level deviation (within plan latitude)

**voices.tenant_id type — UUID FK, not TEXT.** 06.7-PATTERNS.md sketched `tenant_id TEXT NOT NULL` but explicitly flagged it a guess ("Likely TEXT, but verify; if the schema uses UUID or a FK, match it"), and the plan's must_haves truth #3 + Task 2 action mandate matching the existing tenant identity convention. Grepping every prior migration showed the convention is unambiguously `UUID NOT NULL REFERENCES ai_gateway.tenants(id)` (api_keys 0002, usage_counters 0006, billing_events 0010, audit_log 0003) over `tenants.id UUID PK`. The auth context stringifies the UUID for transport (`apikey.go:165`) but the persisted/joined identity is UUID everywhere. Chose UUID FK + ON DELETE CASCADE for referential integrity, tenant-deletion cleanup, and billing-reporting consistency. This is the plan-authoritative choice, not a deviation from intent.
