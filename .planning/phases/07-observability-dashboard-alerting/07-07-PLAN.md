<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Next.js server (RSC + route handlers) | The only code the operator's browser runs; must never carry the admin key |
| Next.js server → gateway /admin/* | Server-side fetch carrying X-Admin-Key |
| Better Auth → its own DB | Session/credential storage, isolated from ai_gateway |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-24 | Information Disclosure | gateway X-Admin-Key | mitigate | RESEARCH.md Anti-Pattern — `GATEWAY_ADMIN_KEY` is read ONLY inside `src/app/api/gateway/[...path]/route.ts` (a server route handler). An acceptance criterion greps the whole `src/` tree and asserts the key appears in exactly that one file; no `NEXT_PUBLIC_` exposure, no client-component reference. The browser only ever sees `/api/gateway/*` relative paths. |
| T-07-25 | Spoofing | unauthenticated dashboard access | mitigate | `src/middleware.ts` runs `getSessionCookie` on every route except `/login` + `/api/auth` and redirects unauthed requests to `/login`. Better Auth `emailAndPassword` gates the ~4 operator accounts. |
| T-07-26 | Tampering | Better Auth DB colliding with ai_gateway | mitigate | Pitfall 7 — `src/lib/db.ts` connects via `DASHBOARD_DATABASE_URL`, a DB/schema isolated from `ai_gateway`. An acceptance criterion asserts no `ai_gateway` connection string appears in the dashboard. Better Auth's user/session/account tables never pollute the gateway schema. |
| T-07-27 | Elevation of Privilege | over-broad Better Auth plugins | mitigate | CONTEXT.md locks ONLY `emailAndPassword` — the converseai-v4 `organization`/`2FA`/`admin` plugins are stripped. No org/role escalation surface in a 4-admin internal tool. |
| T-07-28 | Information Disclosure | secrets in the client bundle | mitigate | `BETTER_AUTH_SECRET`, `GATEWAY_ADMIN_KEY`, `DASHBOARD_DATABASE_URL` are server-only env vars (no `NEXT_PUBLIC_` prefix). `.env.example` documents them as deploy-time Portainer stack vars, never committed. |
</threat_model>
