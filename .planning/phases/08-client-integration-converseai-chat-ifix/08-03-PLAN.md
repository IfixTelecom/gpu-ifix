<objective>
Deliver the Chat Ifix integration smoke test — `scripts/integration-smoke/smoke-chat-ifix.py` — its report schema, a committed real WhatsApp audio fixture, and the recorded baseline it gates against. The script is the `pod/smoke/smoke.py` `run_whisper()` shape, retargeted to the gateway `/v1/audio/transcriptions` multipart endpoint with the `chat-ifix` tenant key. Unlike the pod smoke (which only checks Whisper returned 200 + non-empty text), this script compares transcription **latency AND quality** against a recorded baseline and gates both within ±10% (SC2).

Purpose: INT-02 requires validating that Chat Ifix's WhatsApp-audio Whisper transcription works through the gateway at equivalent latency + quality to the prior direct integration. Chat Ifix's transcription is implemented in the `campanhas-chatifix` sibling repo's backend; this gpu-ifix script is the gateway-side contract evidence the Phase 8 HUMAN-UAT runs.
Output: `scripts/integration-smoke/smoke-chat-ifix.py`, `scripts/integration-smoke/chat-ifix-report-schema.json`, `scripts/integration-smoke/fixtures/README.md`, `scripts/integration-smoke/fixtures/whatsapp-sample.ogg`, `scripts/integration-smoke/fixtures/whatsapp-sample.baseline.json`, a `.gitignore` carve-in so the committed fixture is tracked.

## Planner decisions (surfaced by 08-PATTERNS.md "No Analog Found")

**Audio fixture storage — COMMIT A SHORT REAL CLIP.** Per 08-PATTERNS.md, `pod/smoke/fixtures/` commits no binaries because an 8-min WAV is ~15 MB. Phase 8 needs a *real* speech sample (synthetic noise cannot validate transcription quality). Decision: commit a SHORT real WhatsApp voice clip (5-15 seconds of clear Portuguese speech — a WhatsApp Opus/OGG file is ~10-60 KB at that length, small enough for git). This keeps the smoke test self-contained and CI-runnable with no MinIO dependency. The repo `.gitignore` only excludes `pod/smoke/fixtures/*.wav|*.mp3` — `scripts/integration-smoke/fixtures/` is unaffected, but Task 3 adds an explicit `!scripts/integration-smoke/fixtures/whatsapp-sample.*` allow-rule for safety.

**Transcription-quality gate — WORD ERROR RATE vs the baseline transcript.** No prior smoke compares transcription quality. Decision: compute word error rate (WER) between the live transcript and the committed reference transcript, after normalizing both (lowercase, strip punctuation, collapse whitespace). Quality gate passes when WER <= 0.10 (within 10% — i.e. <=10% of reference words wrong). Latency gate passes when `live_latency_s <= baseline_latency_s * 1.10`. Both mirror the `apply_gates()` "compute metric -> boolean gate" shape from `pod/smoke/smoke.py` lines 315-327.
