## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → gatewayctl → gateway DB | The seed script runs with the gateway DB DSN and mints real tenant identities + API keys that grant access to the gateway `/v1/*` endpoints |
| script stdout / stderr → operator terminal → log files | Raw API keys are emitted to stdout once; stderr (`log()`) may be redirected to a file by the operator |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | raw tenant + admin API keys printed by `provision-tenants.sh` | mitigate | Raw keys are captured into shell variables and emitted ONLY via a stdout final-instructions heredoc — never passed to `log()` (which writes to stderr and may be redirected to a file). Mirrors the gatewayctl `key create` / `admin-key create` secret-once discipline (key.go lines 85-95). The script header documents that keys are shown once and must be copied into Portainer stack env vars, never committed. |
| T-08-02 | Elevation of Privilege | re-running the script sprays duplicate API keys | mitigate | `gatewayctl key create` / `admin-key create` are non-idempotent; the script guards both behind an explicit `--mint-keys` opt-in flag. The default re-run path only runs the idempotent `tenant create` and logs an instruction to pass `--mint-keys` exactly once. This prevents accumulation of orphan keys that widen the auth surface. |
| T-08-03 | Tampering | a typo'd or unexpected gatewayctl failure silently leaves the DB half-provisioned | mitigate | The tenant-create step matches ONLY the literal `already exists` stderr substring as the success-on-exit-1 signal; any other non-zero exit prints the captured stderr and exits 1, so a real failure halts the script instead of being swallowed. `set -euo pipefail` is set. |
| T-08-04 | Repudiation | no record of which operator provisioned the tenants | accept | The `audit_log` / row `created_at` timestamps on `tenants` + `api_keys` (Phase 2 schema) capture provisioning provenance at the DB level; the operator-attribution sign-off lives in the Phase 8 HUMAN-UAT sheet (plan 08-04). No new mitigation needed in the seed script. |
</threat_model>
