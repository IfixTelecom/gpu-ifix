## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Information Disclosure | the `converseai` tenant key embedded as a default in the smoke script | mitigate | The key is supplied ONLY via `--api-key` or the `SMOKE_API_KEY` env var — there is NO committed default. `parse_args()` calls `ap.error(...)` and exits non-zero when no key is provided, so the script cannot run (or be committed) with a baked-in credential. 08-PATTERNS.md "Secret-once stdout discipline". |
| T-08-06 | Information Disclosure | the smoke report JSON leaking the tenant key | mitigate | `report-schema.json` `target` object carries `gateway_url` + `tenant` slug ONLY — the api key is not a schema field, and `additionalProperties: false` rejects any report that tries to add it. The report is safe to attach to the HUMAN-UAT evidence sheet. |
| T-08-07 | Tampering | a gateway error envelope (401/429/503) being silently treated as a pass | mitigate | Every request captures `status_code` + a truncated `raw_error_body` on non-200; `apply_gates()` keys `chat_ok` off `status_code == 200`, so an auth/quota/upstream failure flips the gate and produces a distinct non-zero exit code the HUMAN-UAT asserts on. |
| T-08-08 | Spoofing | the script trusting an unauthenticated/wrong endpoint | accept | The script targets the operator-supplied `--gateway-url`; TLS/cert validation is `httpx` default-on. Pointing it at the wrong host is an operator error caught by the HUMAN-UAT step that checks the dashboard shows traffic under the `converseai` tenant. No new mitigation in scope. |
