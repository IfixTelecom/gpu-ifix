## Round 2 — `aria2c -x 16 --continue` multi-stream download PASS, but llama load FAIL on b9128

**Pod:** offer 8966501 (Iceland, $0.40/h).

**Onstart v3 changes:**
- Replaced `mc cp` with `aria2c --max-tries=10 --retry-wait=5 --max-connection-per-server=16 --split=16 --min-split-size=1M --continue=true` using presigned URL via `mc share download --expire=2h`.
- Fixed earlier bug where `grep -oE 'https://[^ ]+' | head -1` picked the unsigned `URL:` line instead of the signed `Share:` line (caused initial HTTP 403). Replaced with `awk -F': ' '/^Share/{print $2}'`.
- `exec /app/llama-server` foreground (stdout → vast logs) instead of `nohup ... &` to capture llama log directly.

**Result:** 16 GB downloaded successfully + sha256 MATCH (a7cbd3ec...). Aria2c 16-parallel HTTP Range chunks completed where `mc cp` single-stream failed twice.

**Llama-server then FAILED to load Qwen3.6 GGUF:**
```
llama_model_load: error loading model: missing tensor 'blk.64.ssm_conv1d.weight'
llama_model_load_from_file_impl: failed to load model
common_init_from_params: failed to load model '/weights/qwen/model.gguf'
main: exiting due to model loading error
```

**Root cause:** Qwen3.6 27B uses **hybrid Transformer + SSM (State Space Model) architecture** (Mamba-style 1D conv layers). Build `b9128` (commit `856c3adac`, May 13 2026) predates upstream SSM support for Qwen3.6 specifically. Issues 23121 ("server: fix prompt-cache reuse for hybrid/recurrent models") and 22384 ("server: fix context checkpoint restore for hybrid/recurrent models (DeltaNet/Mamba)") merged 2026-05-16 — AFTER b9128.

GitHub issues 22878 and 23210 confirm Qwen3.6-27B LOADS in newer builds (b8684+ for one user, b9186 for another) — only b9128 too old.

Instances 36948435, 36949089 destroyed. Cumulative cost ~$0.92.

### Implications for Phase 6.6 plan structure

**Strategy B-revised (per `06.6-SPIKE-dind-privileged.md` Decision Re-evaluation)** stays viable via supervisord co-location (NOT DinD). Plan 06.6-04 onstart pattern:

```bash
# Install supervisord
apt-get update && apt-get install -y supervisor

# Download model weights (aria2c multi-stream) — Qwen3.6
mkdir -p /weights/qwen
PRESIGN=$(mc share download --expire=2h ifix/${MINIO_BUCKET}/${PRIMARY_QWEN_MODEL_KEY} | awk -F': ' '/^Share/{print $2}')
aria2c --max-tries=10 --max-connection-per-server=16 --split=16 --continue=true --dir=/weights/qwen --out=model.gguf "$PRESIGN"
sha256sum /weights/qwen/model.gguf | awk '{print $1}' | { read actual; [ "$actual" = "$PRIMARY_QWEN_MODEL_SHA256" ]; }

# Supervisord config: 4 services
cat > /etc/supervisor/conf.d/services.conf <<EOF
[program:llama]
command=/app/llama-server --host 0.0.0.0 --port 8000 -m /weights/qwen/model.gguf -ngl 99 -np 2 --ctx-size 16384 --jinja
autostart=true autorestart=true stdout_logfile=/dev/stdout stderr_logfile=/dev/stderr

[program:speaches]
command=/app/speaches-server ...
...

[program:infinity]
command=infinity_emb ...
...

[program:dcgm]
command=/usr/bin/dcgm-exporter ...
...
EOF

exec /usr/bin/supervisord -n -c /etc/supervisor/supervisord.conf
```

`exec supervisord` as PID 1 → 4 child processes share network namespace, GPU, filesystem. Vast.ai port mappings expose each service's port externally.

But — Speaches and Infinity images bring own binaries + python deps. To run them as supervisord child processes inside `llama.cpp:server-cuda-b9191`, we'd need to install them via apt/pip in onstart. Alternative: build a custom multi-stage image bundling all 4 (violates D-08-B "no custom image" rule from CONTEXT.md, but is the cleanest engineering path).

This is a Phase 6.6 replan decision to be locked in WAVE0-GATES.md Decision 2 / 7.
