**1. [Rule 3 - Blocking] LoaderAdapter signature mismatch with real upstreams.Loader**

- **Found during:** Task 1 (adapter interface declaration)
- **Issue:** Plan body specified `LoaderAdapter.OverrideTier0(role, url string) error` and `RestoreTier0(role string)` — but the real `upstreams.Loader.OverrideTier0` is **void** (no error). Forcing the interface to require `error` would force Plan 06.6-06b to introduce an artificial error path on a method that never legitimately fails (the operation is a single atomic.Pointer.Store).
- **Fix:** Declared the interface signatures to match the real Loader contract — `OverrideTier0(role, url string)` (void) + `RestoreTier0(role string)` (void). `Refresh(ctx context.Context) error` retained the error because Loader.Refresh actually re-scans the DB and can fail. Documented as `key-decisions[0]` in frontmatter.
- **Files modified:** `gateway/internal/primary/lifecycle.go`
- **Verification:** `go build ./...` clean; the reconciler call sites in markReady / startDrain / closeLifecycle / recoverOpenLifecycle do NOT check return values on OverrideTier0/RestoreTier0 (the void interface matches usage exactly).
