## Accomplishments

### Task 1: `gateway/internal/primary/schedule.go` + `schedule_test.go`

- Implemented `ScheduleRule` immutable struct (resolved Timezone, UpHour, DownHour, Days, GraceRampDownS, ProvisionLeadS, Disabled).
- `ParseScheduleEnv(cfg) (ScheduleRule, error)` is **fail-fast on invalid timezone** (Pitfall #4 RESEARCH.md:438-443) — operator gets a boot-time error instead of silent UTC fallback that would shift the window by 3+ hours.
- `IsInPeak(now)` implements the **reviews consensus action #5 overnight wrap day-filter**: when `(UpHour >= DownHour) AND now.Hour() < DownHour`, the method consults `Days[(weekday-1+7)%7]` (yesterday's bit, the wrap originator) rather than today's bit. The 3 wrap-semantics tests prove this:
  - `TestIsInPeak_OvernightWrap_MonOnly_TueEarlyIsInPeak` — mon-only enabled + UP=22 DOWN=8 + Tue 02:00 BRT → `true` (Monday wraps).
  - `TestIsInPeak_OvernightWrap_TueOnly_TueEarlyIsNotInPeak` — tue-only enabled + same → `false` (Monday off, Tuesday's own UP=22 hasn't fired).
  - `TestIsInPeak_OvernightWrap_TueOnly_TueLateIsInPeak` — tue-only + Tue 23:00 → `true` (Tuesday's bit applies; hour >= UpHour).
- `ShouldBeProvisioned(now)` is the **NEW method per reviews consensus action #8**: returns true throughout `IsInPeak` AND for `ProvisionLeadS` seconds before the next "up" transition. Plan 06.6-06a `reconciler.evaluateAsleep` will use this (not `IsInPeak`) for the provisioning decision so the pod is Ready by UpHour given a 25–30min cold-start.
- `NextTransition(now)` walks 8 days forward (bounded to guarantee termination on pathological configs) returning the next `(time, "up"|"down")` for operator visibility (`gatewayctl primary schedule show`) and for `ShouldBeProvisioned`'s lead-time arithmetic.
- Kill-switch: `Disabled=true` short-circuits both `IsInPeak` and `ShouldBeProvisioned` to `false` regardless of time. Default `Disabled=true` per WAVE0-GATES Decision 5 (operator manual flip after UAT GREEN).
- **25 unit tests** under `-race`, all green:
  - 6 `ParseScheduleEnv_*` (defaults, tz fail-fast, day-filter weekdays default, custom CSV, invalid-day-ignored, ProvisionLeadSeconds respected).
  - 12 `IsInPeak_*` (default rule, before up, after down, overnight Tuesday late, overnight before up, 3 wrap-day cases, weekend excluded, kill-switch, **DST coverage via America/New_York spring-forward fixture**).
  - 5 `ShouldBeProvisioned_*` (active window, 30min lead before UpHour, beyond-lead 60min before, custom 60min lead, kill-switch override).
  - 3 `NextTransition_*` (before peak → up, during peak → down, weekend skipped → next Monday up).

### Task 2: `gateway/internal/primary/fsm.go` + `fsm_test.go`

- `State int32` enum with 5 states: `StateAsleep`, `StateProvisioning`, `StateReady`, `StateDraining`, `StateDestroying`. `allStates` slice + `String()` switch covering all 5 + `"unknown"` fallback.
- `ErrInvalidTransition` sentinel returned by `Transition` on CAS failure — divergence from `gateway/internal/emerg/fsm.go` (which is silent noop) is intentional: the reviews #8 pre-warm flow needs explicit error so the reconciler can drive retries on the next tick.
- `FSM` struct: `atomic.Int32` state + `atomic.Int64` enteredAt (nanoseconds) + `onChange` callback + `stateChangeWriter` audit hook. `State()` and `EnteredAt()` are lockless `atomic.Load`.
- `NewFSM(writer, onChange)` constructor: both args nilable for early-boot wiring. Initialises state to `StateAsleep`.
- `Transition(from, to, at, reason) error`: same-state filtered as noop (no error, no callback, no writer call); `CompareAndSwap` guards the state transition; on success → `commitTransitionSideEffects` fires the writer + onChange.
- `SetState(to, at, reason)`: unconditional CAS loop (used by leader-recovery resume).
- `stateChangeWriter` interface intentionally accepts `event any` so primary/ does not depend on `internal/audit` — Plan 06.6-06+ reconciler wraps a typed adapter.
- **10 unit tests** under `-race`, all green:
  - String round-trip (all 5 states + unknown).
  - Initial state is `StateAsleep`.
  - Happy-path transition (state + EnteredAt updated).
  - Stale-from returns `ErrInvalidTransition`; state unchanged; no callback.
  - Same-state noop (no error, no callback).
  - onChange captures (from, to, at, reason) verbatim across 2 transitions.
  - SetState forces transitions; same-target SetState is noop.
  - **100-goroutine CAS race** → exactly 1 ok return + 99 `ErrInvalidTransition` + 1 callback fire.
  - Writer receives `WriteStateChange("fsm_transition", ...)` only after successful CAS — noop and stale paths emit zero writes.
  - Concurrent read-during-transition race-detector smoke (lockless `State()` does not race with CAS).
