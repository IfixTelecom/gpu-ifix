### Engine + modelo LLM

- **D-03:** Engine LLM = **`ghcr.io/ggml-org/llama.cpp:server-cuda-b9191`** (NÃO b9128 — Wave 0 spike empirically validated b9128 fails Qwen3.6 SSM tensor load). Image SHA: `sha256:cb375311f4170bb1aa18840e946f64f99e6094b90bde69dcb6e0a62a183d7ba3`.
- **D-04:** Modelo LLM = **Qwen3.6-27B Q4_K_M GGUF** (`unsloth/Qwen3.6-27B-MTP-GGUF` upstream, file `Qwen3.6-27B-Q4_K_M.gguf`). Already uploaded em MinIO:
  - Key: `qwen3.6-27b-Q4_K_M/v1.0.0/model.gguf` (17.106 GB)
  - Sha256: `a7cbd3ecc0e3f9b333edee61ae66bc87ed713c5d49587a8355814722ed329e0f`
  - Modo: **text-only** (multimodal vision endpoint = phase futura).
- **D-04-alt:** Jinja tool-calling para Qwen3.6 = **B1 embedded LOCKED** (Wave 0 spike PASS Round 3). `PrimaryQwenJinjaKey` default = empty string. `--jinja` flag sozinho (sem `--chat-template-file`) extrai PEG-native parser do GGUF embedded chat_template. Tool-calling validado: `tool_calls[0].function.arguments={"location":"São Paulo"}`. Override via env vars permitido pra rota futura.

### Health probing — Drop health-bridge

- **D-05:** **Drop pod/health-bridge custom Go service**. Phase 3 gateway probe (`ProbeIntervalSeconds=10`) já roda active healthchecks com latency tracking + breaker FSM. Duplicação eliminada.

### Orchestration approach — supervisord

- **D-06:** **supervisord co-located LOCKED** (Wave 0 DinD REJECTED). PID 1 = supervisord, 4 child processes:
  ```
  /etc/supervisor/conf.d/services.conf
    - [program:llama]   exec /app/llama-server --host 0.0.0.0 --port 8000 -m /weights/qwen/model.gguf -ngl 99 -np 2 --ctx-size 16384 --jinja
    - [program:speaches] exec /app/speaches ...
    - [program:infinity] exec infinity_emb ...
    - [program:dcgm]    exec /usr/bin/dcgm-exporter ...
  ```
  Children share container's network namespace (localhost ports between siblings), GPU device, filesystem.
- **D-06.1 (NEW Wave 0):** **Strategy α — Custom multi-stage image LOCKED.** `pod/primary/Dockerfile` build:
  ```dockerfile
  FROM ghcr.io/speaches-ai/speaches:0.9.0-rc.3-cuda-12.6.3@sha256:5c62... AS speaches-stage
  FROM michaelf34/infinity:0.0.77@sha256:11e8... AS infinity-stage
  FROM nvcr.io/nvidia/k8s/dcgm-exporter:4.5.3-4.8.2-distroless@sha256:60d3... AS dcgm-stage
  FROM ghcr.io/ggml-org/llama.cpp:server-cuda-b9191@sha256:cb37... AS final
  RUN apt-get install -y supervisor aria2 ca-certificates
  COPY --from=speaches-stage /app /app/speaches-bin
  COPY --from=infinity-stage /app /app/infinity-bin
  COPY --from=dcgm-stage /usr/bin/dcgm-exporter /usr/bin/dcgm-exporter
  COPY supervisord.conf /etc/supervisor/conf.d/services.conf
  ```
  GHA workflow builds + pushes `ghcr.io/ifixtelecom/converseai-primary-pod:vX` (semver bumped per release). Onstart curto (~1KB): aria2c weights + exec supervisord.
  - **Revisits Phase 6 D-08-B "no custom image" principle** intentionally for Phase 6.6 ONLY (4-service co-location requires it). Phase 6 emerg pod remains custom-image-free.
- **D-06.2 (NEW Wave 0):** **No explicit GPU offload check in onstart.** Gateway Phase 3 probe loop (ProbeIntervalSeconds=10, P95 latency breaker FSM) detects CPU-fallback degradation naturally — high latency = breaker opens = fallback chain redirect. Existing mechanism, no Plan 06.6-04 instrumentation overhead. Acceptable 5-10s detection delay vs immediate-fail strict check.
- **D-06.3 (NEW Wave 0):** **VRAM budget tight fit, full quality models.** Qwen3.6 27B Q4_K_M (~17GB) + KV cache @16k (~2GB) + Whisper-Large-v3 (~3GB) + BGE-M3 full (~2GB) = ~24GB. Zero margin on RTX 4090. Risk OOM under peak KV growth ACCEPTED — researcher should validate `--keep` / `--n-keep` flags to cap KV growth. If empirical OOM observed in Live UAT (Plan 06.6-11), fallback path = downgrade Whisper-Medium + BGE-M3 INT8 (override env vars `PRIMARY_WHISPER_MODEL` / `PRIMARY_EMBED_MODEL`).

### Lifecycle trigger — Schedule-based via env vars

- **D-08:** **Schedule-based via env vars** (não 24/7, não pure on-demand). Pod up em peak hours, down em off-hours. Custo estimado ~$130/mo (~14h × 22 dias × $0.40).
- **D-08.1:** Env vars (nomes finais research-driven, sugestão inicial):
  - `PRIMARY_POD_SCHEDULE_TIMEZONE=America/Sao_Paulo`
  - `PRIMARY_POD_SCHEDULE_UP_HOUR` (e.g., 8)
  - `PRIMARY_POD_SCHEDULE_DOWN_HOUR` (e.g., 22)
  - `PRIMARY_POD_SCHEDULE_DAYS` (CSV `mon,tue,wed,thu,fri`)
  - `PRIMARY_POD_SCHEDULE_GRACE_RAMP_DOWN_SECONDS` (drain inflight antes de destroy)
  - `PRIMARY_POD_SCHEDULE_DISABLED` (operator kill-switch)
  - `PRIMARY_POD_SCHEDULE_PROVISION_LEAD_SECONDS` (pre-warm offset, e.g., 1800 = 30min before UP_HOUR)
- **D-08.2:** Off-hours behavior = apps caem em Phase 3 fallback chain (OpenRouter LLM + OpenAI Whisper STT + OpenAI embed). Já validado functionally.
- **D-08.3 (UPDATED Wave 0):** **Extend `gateway/internal/emerg/reconciler.go`** (NÃO novo file). Code reuse via shared FSM + cooldown logic + Vast client wiring. Shared helpers já extracted by Plan 06.6-02 (gateway/internal/vastutil/).

### Soak gate + cold-start budget (NEW Wave 0)

- **D-09 (NEW):** **`PRIMARY_POD_SCHEDULE_DISABLED=true` default** at PR2 deploy. Operator flip manual to `false` only after Live UAT GREEN (Plan 06.6-11). Same pattern as Phase 6 PR2.
- **D-10 (NEW):** **`PRIMARY_PROVISION_COLDSTART_BUDGET_SECONDS = 2400`** (40 min). Realistic 15-25 min; budget 40 min = generous margin (accommodates slow inet hosts, host cold image pull, supervisord init fragility). Reconciler treats > 40 min as provision failure → enter cooldown.

### Weight cache strategy

- **D-11:** Weights baixados toda lifecycle via **aria2c multi-stream** (`aria2c --max-tries=10 --retry-wait=5 --max-connection-per-server=16 --split=16 --min-split-size=1M --continue=true`). Wave 0 validated: `mc cp` single-stream falha `unexpected EOF` em WAN long-lived TCP; aria2c chunks parallel + auto-retry resolve. Files: Qwen3.6 (~16GB) + Whisper (~3GB) + BGE-M3 (~2GB) = ~21GB total via aria2c parallel.
- **D-11.1:** Presigned URLs via `mc share download --expire=2h ... | awk -F': ' '/^Share/{print $2}'` (Wave 0 spike caught bug: `grep | head -1` picks unsigned `URL:` line; awk-by-prefix is correct).
- **D-11.2:** Vast persistent volumes + host pin = **deferred** (arch follow-up; mesmo gap SC-2 Phase 6).

### Operator debug SSH

- **D-12:** **Reusar `POD_DEBUG_SSH_PUBLIC_KEY`** env var (Phase 6 pattern). Production: env vazio (least-privilege).
