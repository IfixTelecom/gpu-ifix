## Decision 2 — Orchestration Strategy (D-06)

**Strategy B-revised via supervisord — LOCKED.** DinD REJECTED.

**Rationale:** Spike Task 1 confirmed DinD impossible on Vast.ai hosts even with `--privileged:1` envelope:
- `--privileged:1` accepted by Vast.ai env (passes through to docker container).
- dockerd boots inside container, but `iptables=false + bridge=none` workaround required (host nf_tables manipulation blocked).
- Even with workaround, **overlayfs mount/unmount in nested namespace fails** — Linux kernel namespace limitation, not bypassable.

**Strategy B's INTENT preserved** (4 services co-located in 1 pod for low intra-pod latency, GPU sharing, shared filesystem) **via supervisord as PID 1**:

```
PID 1 = supervisord
  ├─ llama-server  (Qwen3.6 LLM, port 8000)
  ├─ speaches      (Whisper STT, port 8001)
  ├─ infinity      (BGE-M3 embeddings, port 8002)
  └─ dcgm-exporter (GPU metrics, port 9400)
```

All children share container's network namespace, GPU device, filesystem. Vast.ai env exposes each port: `{"-p 8000:8000":"1","-p 8001:8001":"1","-p 8002:8002":"1","-p 9400:9400":"1"}`.

**VRAM budget on RTX 4090 (24GB):**
- llama.cpp Qwen3.6 27B Q4_K_M: ~17GB (model) + ~2GB KV cache @ 16k ctx = ~19GB
- Speaches Whisper-Large-v3: ~3GB
- Infinity BGE-M3: ~2GB
- DCGM-exporter: ~0 (CPU-only)
- **Total: ~24GB** — tight fit. Consider Whisper-Medium-v3 (~1.5GB) or quantized BGE-M3 if overflow. Plan 06.6-04 must surface VRAM math + offer override env vars (`PRIMARY_WHISPER_MODEL`, `PRIMARY_EMBED_MODEL`).

### Implementation strategy — α (custom image) vs β (runtime install)

Decision deferred to plan replan (Plan 06.6-02 / 06.6-04). Two candidates:

- **α (custom image):** Build multi-stage image bundling all 4 binaries + supervisord. Violates Phase 6.6 CONTEXT.md D-08-B ("no custom image") but engineering-cleanest. Single SHA pin, no runtime apt install in onstart.
- **β (runtime install):** Use `ghcr.io/ggml-org/llama.cpp:server-cuda-b9191` as base. Onstart `apt install supervisor` + download speaches/infinity/dcgm binaries (apt or pip or pre-built tarballs from MinIO) + write supervisord config. Respects D-08-B but adds onstart fragility (apt cache, network reliability, version drift).

**Recommendation for replan:** Lock α (custom image) — D-08-B was a Phase 6 emerg-pod convention that doesn't bind 06.6 specifically. A small `Dockerfile` in `pod/primary/Dockerfile` that does multi-stage `FROM speaches AS speaches-binaries; FROM infinity AS infinity-binaries; FROM llama.cpp:server-cuda-b9191 + COPY --from + apt install supervisor + supervisord.conf` is ~30 lines and far simpler than runtime install fragility. GHA workflow builds + pushes to ghcr.io/ifixtelecom/converseai-primary-pod:vX.

Operator decision needed at replan time. Not Wave 0 blocking — both viable.

## Decision 6 — Cold-Start Budget

**`PRIMARY_PROVISION_COLDSTART_BUDGET_SECONDS = 2400` (40 minutes).**

Per CONTEXT.md Open Question 7. Breakdown:
- Image pull (cached host best-case): 0s | (uncached worst-case): ~3 min
- Aria2c download Qwen3.6 16GB GGUF: empirically ~3 min on Iceland (3.5 Gbps inet) host; budget 5 min for slower hosts
- (If α custom image) Whisper model download (~3GB cached in image) + Infinity model HF cache (~2GB pre-warmed): 0s
- (If β runtime install) supervisord install (~10s apt) + speaches binary download (~30s) + infinity pip install (~2 min) + dcgm-exporter binary (~10s): ~3 min
- llama-server model load (Qwen3.6 27B → VRAM): ~30-60s
- Supervisord process spawn + health check (4 services): ~30s
- Buffer: 5 min

**Total realistic: 15-25 min. Budget 40 min = generous margin** (accommodates slow inet hosts, host cold image pull, supervisord install fragility). Reconciler treats > 40 min as provision failure → enter cooldown.

Phase 6 emergency pod (single LLM) measured 21 min cold-start in Live UAT (Spain ES, 14 MB/s WAN). Primary pod with 4 services has more downloads but parallelizable via aria2c + parallel Speaches/Infinity startup.
