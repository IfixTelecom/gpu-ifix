<objective>
Wave 2: Move model rewriting from request-edge (`models.Handler` in main.go) INTO each per-upstream Director, so the rewrite happens AFTER the dispatcher has selected which upstream gets the request. This is the load-bearing behavioral fix that closes OR-FIX, STT-OAI-FIX, and EMBED-OAI-FIX.

Three directors need parallel changes (same shape, three implementations):
1. `BuildOpenRouterDirector` — gains resolver+upstreamName; folds model rewrite into the existing `injectProviderOrder` body-rewrap pass.
2. `BuildOpenAIWhisperDirector` — currently passes multipart body through untouched (Phase 02 deferral); gains a NEW multipart-aware `model` form-field rewriter that streams audio bytes via `io.Copy` (never decodes binary as text).
3. `BuildOpenAIEmbedDirector` — currently hard-codes `text-embedding-3-small`; refactor to source via resolver (consistency cleanup). `dimensions=1024` stays hard-coded (BGE-M3 parity invariant).

main.go wiring: delete the 2 `models.Handler` wraps for chat/embed handlers (per Option (a) in 06.9-PATTERNS.md — PICK (a) per planning_context); pass `resolver + upstreamName` into each `Build*Director` constructor at its wire-up site.

Purpose: After this plan, gateway request `POST /v1/chat/completions {"model":"qwen"}` with `local-llm` breaker OPEN forwards `{"model":"qwen/qwen3.5-27b", "provider":{"order":["novita"], ...}, ...}` to OpenRouter — proven by 3 new unit tests + Plan 05 integration test.

Output: 6 source files modified + 3 director test files extended. `cd gateway && go test ./internal/proxy/... -count=1 -race` PASS including 3 new rewrite tests. Build succeeds.
</objective>
