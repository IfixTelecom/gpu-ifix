<objective>
Wave 2 (parallel to Plan 03): Add two operator-surface improvements that Plan 06 (live UAT) depends on:

1. **Config fail-fast on URL convention** (D-07): reject `UPSTREAM_LLM_OPENROUTER_URL` / `UPSTREAM_STT_OPENAI_URL` / `UPSTREAM_EMBED_OPENAI_URL` ending in `/v1` at boot time. Prevents the silent double-`/v1` 404 bug that masked OR-FIX for months. Clear error message names the offending var + suggests correct value (`https://openrouter.ai/api`).

2. **gatewayctl breaker subcommand** (`breaker force-open` / `breaker force-close` / `breaker list`): operator-driven deterministic breaker state, required by Plan 06 HUMAN-UAT to drive S1-S3 scenarios without depending on probe-failure timing or primary-pod cycling.

3. **gatewayctl model-alias subcommand** (`model-alias list` / `model-alias set`): operator override path that replaces the deprecated `UPSTREAM_LLM_OPENROUTER_MODEL` env var. Schema row is the single source of truth (D-06); CLI lets operator tweak per-upstream targets without re-deploying. Mirrors the `upstreams` subcommand shape (gateway/cmd/gatewayctl/upstreams.go).

Purpose: Plan 06 needs both the fail-fast (so misconfig is caught loudly) AND the breaker control (so live UAT is deterministic). This plan is independent of Plan 03 — different files, different surface, parallel-safe.

Output: 5 new files + 2 modified files. Boot fail-fast verified via unit test. `gatewayctl breaker force-open openrouter-chat` and `gatewayctl model-alias list` execute against the local dev stack in under 100ms each.
</objective>
