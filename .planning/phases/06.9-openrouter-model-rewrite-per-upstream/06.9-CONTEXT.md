## Phase Boundary

**Fix the gateway dispatcher → tier-1 director model-name rewriting gap.** When the primary pod is asleep + breaker OPEN, gateway must dispatch chat requests to the configured tier-1 upstream (OpenRouter for LLM, OpenAI for STT/Whisper, OpenAI for embed) WITH the per-upstream model slug, not the gateway-internal alias. Today the dispatcher forwards `{"model":"qwen"}` literal to OpenRouter, which returns HTTP 404 because `qwen` is not a valid OpenRouter slug. Wave 0 Gate A (Phase 03, 2026-04-20) anticipated this and defined `UPSTREAM_LLM_OPENROUTER_MODEL=qwen/qwen3.5-27b` as the env var to set — but Plan 03-06 implementation never consumed it. Same shape almost certainly affects `UPSTREAM_STT_OPENAI_MODEL` and `UPSTREAM_EMBED_OPENAI_MODEL`.

**Net result after this phase:**
- `POST /v1/chat/completions {"model":"qwen"}` against ai-gateway-dev with `local-llm` breaker OPEN → HTTP 200 with real OpenRouter Qwen 3.5 completion via Novita (or fallback DeepInfra)
- Same shape verified for `openai-whisper` (`model:"whisper-1"` rewrite) and `openai-embed` (`model:"text-embedding-3-small"`)
- Per-upstream model resolution is schema-driven (`model_aliases` PK widened to `(alias, upstream_name)`) — extensible without code changes when new tier-1 providers land
- Regression test in dispatcher asserts model rewrite per resolved-upstream
- Phase 03 SC-1 live UAT closes (Phase 02 step 7 chat E2E + Phase 05 SC-1 full overflow cascade)

**Not this phase:**
- Switching primary LLM to Qwen 3.6 on OpenRouter — Qwen 3.6 not yet published on OpenRouter (per PROJECT.md decision row, drift accepted)
- Changing OpenRouter provider pin away from Novita (Wave 0 Gate A decision holds)
- Phase 03 SC-1 Sentry breadcrumb inspection (operator UI step, separate)
- Adding new tier-2 providers (out of scope; future SEED)
- Per-tenant model overrides (tenant-scoped models orthogonal to per-upstream model rewriting)

### Root cause trace

| Layer | File | Behavior |
|-------|------|----------|
| Dispatcher | `gateway/internal/proxy/dispatcher.go` | Selects `openrouter-chat` correctly when `local-llm` breaker OPEN |
| Model resolver | `gateway/internal/models/rewrite.go` + `gateway/db/migrations/0005_create_model_aliases.sql` | Rewrites alias `qwen` → target `qwen` (single-row PK on `alias`); no per-upstream target |
| Director (base) | `gateway/internal/proxy/director.go:39-62` | `BuildDirector` sets `r.URL.Scheme/Host = upstream.Scheme/Host`; **deliberately keeps `r.URL.Path = /v1/chat/completions` unchanged** (comment: "pod routes mirror gateway 1:1") |
| Director (OpenRouter) | `gateway/internal/proxy/openrouter_director.go:36-77` | Wraps base + injects `Authorization`, `provider.order`, `stream_options`; **does NOT touch body.model** |

`UPSTREAM_LLM_OPENROUTER_MODEL` env var present in:
- `.env.example`, `.env.portainer.dev` (documented for operator)
- `gateway/internal/proxy/interceptor_usage.go:259` (cost lookup only)
- `03-WAVE0-GATES.md` line 88 (operator gate spec)
- `gateway/internal/integration_test/*_test.go` (test fixtures)

NOT in any active wiring path (dispatcher, director, config) on `develop`.

### URL gotcha (already worked around in `/opt/ai-gateway-dev/.env`)

`UPSTREAM_LLM_OPENROUTER_URL` MUST end at `/api` (NOT `/api/v1`). Because `BuildDirector` keeps `r.URL.Path = /v1/chat/completions` unchanged, URL `https://openrouter.ai/api` + path `/v1/chat/completions` resolves to correct upstream `https://openrouter.ai/api/v1/chat/completions`. With Wave 0 Gate A's documented `https://openrouter.ai/api/v1`, you get double-`/v1`. **Already applied in /opt/ai-gateway-dev/.env this session.** Update Wave 0 Gate A doc + .env.portainer.example to match.

## Implementation Decisions

### D-01: Fix approach = Option B (schema-driven per-upstream resolution)

Two options surfaced in SEED-004:
- **A (director-level rewrite)** — read `UPSTREAM_LLM_OPENROUTER_MODEL` env in `openrouter_director.go`, rewrite body.model. Localized but scattered: each upstream director needs identical wiring (3x: openrouter-chat, openai-whisper, openai-embed). Future tier-1 providers repeat the pattern.
- **B (schema-driven)** — widen `model_aliases` PK from `(alias)` to `(alias, upstream_name)`. Resolver picks per-resolved-upstream target. Dispatcher already knows which upstream it dispatched to (logs `upstream=openrouter-chat`). Single Handler change, extensible to N providers.

**Picked B.** Schema-driven, extensible, single source of truth. Phase 07 dashboard needs per-upstream model visibility anyway.

### D-02: Migration strategy — additive, no data loss

Migration `0026_evolve_model_aliases_per_upstream.sql`:
- ALTER TABLE drop existing PK (alias)
- ADD COLUMN `upstream_name TEXT NOT NULL DEFAULT 'local-llm'` (or per-row default based on existing `upstream` column)
- ADD PK (alias, upstream_name)
- INSERT new rows for each (alias, tier-1-upstream) pair (qwen+openrouter-chat → qwen/qwen3.5-27b; whisper+openai-whisper → whisper-1; bge-m3+openai-embed → text-embedding-3-small)
- Backfill existing rows with `upstream_name = '<tier-0-upstream>'` (local-llm/local-stt/local-embed)
- Down migration restores PK on (alias) — accept that the seeded tier-1 rows survive but won't be selected (no breaking change on rollback)

### D-03: Resolver + Handler signature — pass upstream_name through

`models/resolver.go` Cache key changes from `alias string` → `(alias, upstream_name string)`. `models/rewrite.go` Handler receives resolved upstream from dispatcher context (already there per `auditctx.WithUpstream` per Phase 4). Director invocation order matters: model resolution MUST happen AFTER dispatcher picks upstream (otherwise tier-0 model is used when tier-1 was actually selected).

### D-04: 3 directors get the same rewrite-from-resolved-model treatment

- `openrouter_director.go` — model rewrite from resolver (NEW)
- `openai_whisper_director.go` (if exists; verify) — model rewrite from resolver (NEW)
- `openai_embed_director.go` (if exists; verify) — model rewrite from resolver (NEW)

If `openai_whisper_director.go` / `openai_embed_director.go` don't exist yet (i.e., handled by generic BuildDirector), the rewrite belongs in a shared wrapper / pre-director step.

### D-05: Live UAT scenarios — 3 SC live PASS + cascade verification

S1: Primary down → POST `/v1/chat/completions {"model":"qwen"}` → HTTP 200 + real OpenRouter completion (LLM tier-1)
S2: Primary down → POST `/v1/audio/transcriptions` with WAV + `model=whisper` → HTTP 200 + real OpenAI Whisper transcription (STT tier-1)
S3: Embed tier-0 breaker forced → POST `/v1/embeddings {"model":"bge-m3","input":["test"]}` → HTTP 200 + real OpenAI embedding tier-1
S4: Phase 03 SC-1 cascade — re-run Phase 03 HUMAN-UAT scenario, mark PASS
S5: Phase 02 SC-5 step 7 cascade — re-run Phase 02 UAT step 7, mark PASS
S6: Phase 05 SC-1 cascade — re-run vegeta burst + assert tier-1 200s instead of 502s

### D-06: Provider env vars stay where they are

`UPSTREAM_LLM_OPENROUTER_AUTH_BEARER`, `UPSTREAM_LLM_OPENROUTER_PROVIDER_ORDER` stay as-is. `UPSTREAM_LLM_OPENROUTER_MODEL` becomes redundant once schema drives the model name — but keep it as a fallback override (env value wins over schema row if set). Document in Wave 0 Gate A doc revision.

### D-07: URL convention enforcement

Add config validation: `UPSTREAM_LLM_OPENROUTER_URL` MUST NOT end with `/v1` (Director path concat will produce `/v1/v1`). Fail-fast with explicit error on boot. Update Wave 0 Gate A + .env.portainer.example to use `https://openrouter.ai/api` (no `/v1`).
