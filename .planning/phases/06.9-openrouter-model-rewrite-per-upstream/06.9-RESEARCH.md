## Summary

The dispatcher (Phase 03) correctly selects `openrouter-chat` when the `local-llm` breaker is OPEN, but the request body that reaches OpenRouter still carries `{"model":"qwen"}` â€” the gateway-internal alias resolved by `models.Handler` at the request-edge for the **local-llm** tier-0 tokenizer. OpenRouter rejects with HTTP 404 because `qwen` is not a valid OpenRouter slug; the canonical slug is `qwen/qwen3.5-27b` (resolves to `qwen/qwen3.5-27b-20260224`, 6 active providers including Novita). The same gap exists for `openai-whisper` (must rewrite to `whisper-1`) but does NOT exist for `openai-embed` (Plan 03-06 hard-coded `text-embedding-3-small` + `dimensions=1024` in `BuildOpenAIEmbedDirector`). Phase 02 step 7 chat E2E, Phase 03 SC-1, and Phase 05 SC-1 all cascade-close on this fix.

The chosen approach (CONTEXT.md D-01 = Option B) is schema-driven: widen the `model_aliases` PK from `(alias)` to `(alias, upstream_name)` so the resolver can return the per-upstream target after the dispatcher has selected an upstream. This is more invasive than a director-local env-var rewrite but extensible (no code change when a new tier-1 provider lands) and aligns with the Phase 07 dashboard's per-upstream model visibility needs.

**Primary recommendation:** Migration 0026 widens the PK, seeds 3 tier-1 rows, leaves the 3 existing rows backfilled with their tier-0 upstream NAME (not the abstract role). Refactor `resolver.aliases` key from `(alias, role)` to `(alias, upstream_name)`. Move model rewrite from request-edge (`models.Handler`) to the **Director** of each tier-1 proxy so rewrite happens AFTER dispatcher resolution. Add config fail-fast for URL convention. Add integration-test fixtures that assert the forwarded body's `model` field. Bundle config-fail-fast and Wave 0 Gate A doc revision in the same phase.

### PK widening recipe for `model_aliases`

```sql
-- +goose Up
-- +goose StatementBegin
SET search_path = ai_gateway, public;

-- Phase 06.9 â€” widen model_aliases PK from (alias) to (alias, upstream_name).
-- The existing `upstream` column carries the ROLE (llm/stt/embed) â€” keep it
-- for backward-compat queries + audit; add a new `upstream_name` column that
-- carries the actual upstream NAME ('local-llm', 'openrouter-chat', etc.).
-- Backfill `upstream_name` from the existing `upstream` role by mapping
-- role â†’ tier-0 upstream name (the canonical "primary for this role"), then
-- INSERT three NEW tier-1 rows for (qwen+openrouter-chat),
-- (whisper+openai-whisper), (bge-m3+openai-embed).
--
-- Rollback (Down): restore PK=(alias), accept that tier-1 rows survive
-- but stop being selected (no breaking-change risk; resolver lookup keyed
-- on (alias) returns the first row's target by ORDER BY alias = tier-0).

-- Step 1 â€” add column, default to NULL so existing rows are visible
ALTER TABLE ai_gateway.model_aliases
    ADD COLUMN IF NOT EXISTS upstream_name TEXT;

-- Step 2 â€” backfill existing rows with their tier-0 upstream name
--   role='llm'    â†’ upstream_name='local-llm'
--   role='stt'    â†’ upstream_name='local-stt'
--   role='embed'  â†’ upstream_name='local-embed'
UPDATE ai_gateway.model_aliases
SET upstream_name = CASE upstream
    WHEN 'llm'   THEN 'local-llm'
    WHEN 'stt'   THEN 'local-stt'
    WHEN 'embed' THEN 'local-embed'
END
WHERE upstream_name IS NULL;

-- Step 3 â€” enforce NOT NULL post-backfill
ALTER TABLE ai_gateway.model_aliases
    ALTER COLUMN upstream_name SET NOT NULL;

-- Step 4 â€” drop old PK, add new composite PK
ALTER TABLE ai_gateway.model_aliases
    DROP CONSTRAINT IF EXISTS model_aliases_pkey;
ALTER TABLE ai_gateway.model_aliases
    ADD CONSTRAINT model_aliases_pkey PRIMARY KEY (alias, upstream_name);

-- Step 5 â€” seed the 3 tier-1 rows. ON CONFLICT (alias, upstream_name) DO NOTHING
-- makes the migration idempotent across re-runs.
INSERT INTO ai_gateway.model_aliases (alias, upstream, target, upstream_name) VALUES
    ('qwen',    'llm',   'qwen/qwen3.5-27b',          'openrouter-chat'),

    'Phase 06.9 â€” upstream NAME (e.g. local-llm, openrouter-chat). Composite PK with alias. Separate from `upstream` which keeps the role enum for backward-compat audit queries.';
-- +goose StatementEnd

-- +goose Down
-- +goose StatementBegin

-- WARNING: rollback retains the tier-1 seed rows. With PK back to (alias),
-- ORDER BY alias returns the first matching row's target. The current
-- alphabetical sort puts 'local-*' before 'openai-*' / 'openrouter-*'
-- which keeps the tier-0 target as the "winning" row for any non-aware
-- lookup â€” but if a downstream caller picks the wrong row, the operator
```

### Rollback notes

- **No FK references** to `model_aliases` exist; safe to widen PK without touching dependent tables.
- **sqlc regeneration required** after migration lands: `cd gateway && sqlc generate` (the `AiGatewayModelAlias` struct in `internal/db/gen/models.go:123` gains a `UpstreamName string` field; `GetModelAlias` query needs adjustment to accept the composite key OR be replaced).
- **migrate_test.go hard-coded list**: must add `"0026_evolve_model_aliases_per_upstream.sql"` to the slice (line 64 currently ends at `0025_create_voices.sql`).
- **Boot-time idempotency**: stack runs `AI_GATEWAY_MIGRATE_ON_BOOT=true` per dev env â€” the migration applies on next container restart automatically. Backfill step is idempotent (only updates `WHERE upstream_name IS NULL`).
- **No goose hot-reload hook for model_aliases**: resolver uses a 60s Ticker (`resolver.go:18`), so post-migration the gateway picks up new rows within 60s of container restart. There is NO LISTEN/NOTIFY trigger on this table (unlike `upstreams` which has `notify_upstreams_changed` in migration 0009). If operator needs faster propagation in production, the plan can OPTIONALLY add a similar trigger in this migration, but the existing 60s Ticker is acceptable for the immediate fix.

## Resolver Refactor Approach

### Cache key change

Current shape (`resolver.go:26-29` + `:42`):

```go
type aliasKey struct {
    Alias    string
    Upstream string // currently carries ROLE ("llm" | "stt" | "embed")
}

aliases map[aliasKey]string
```

New shape: the `Upstream` field's value semantics change from **role** to **upstream name**. The struct field name stays for diff minimization; the comment is updated and the Refresh loop reads the new `upstream_name` DB column.

### Refresh impact

`resolver.go:60-63` currently:

```go
for _, row := range rows {
    fresh[aliasKey{Alias: row.Alias, Upstream: row.Upstream}] = row.Target
}
```

After sqlc regeneration, `row` gains `row.UpstreamName`. Change to:

```go
for _, row := range rows {
    fresh[aliasKey{Alias: row.Alias, Upstream: row.UpstreamName}] = row.Target
}
```

The 60s Ticker (`resolver.go:74-89`) needs no change. The atomic swap pattern (RWMutex around the map) is preserved.

### Handler signature change

`models.Handler(resolver, upstream, inner)` at `rewrite.go:60` is invoked at `main.go:1213` and `:1216` with the **role** (`"llm"` / `"embed"`):

```go
chatHandler = models.Handler(px.resolver, "llm", chatHandler)
embedHandler = models.Handler(px.resolver, "embed", embedHandler)
```

This is the wrong layer post-fix. **Remove these two calls.** The model rewrite moves into the per-upstream Director, which knows its own upstream name at construction time:

```go
// inside buildOpenRouterChatProxy
Director: proxy.BuildOpenRouterDirector(parsed, u.AuthBearer,
    cfg.UpstreamOpenRouterProviderOrder, cfg.UpstreamOpenRouterAllowFallbacks,
    resolver, "openrouter-chat"), // NEW: resolver + upstream name
```

Inside the Director, fold the rewrite into the existing `injectProviderOrder` body-rewrap pass so we don't allocate twice:

```go
func BuildOpenRouterDirector(upstream *url.URL, authBearer string,
    providerOrder []string, allowFallbacks bool,
    resolver *models.Resolver, upstreamName string) func(*http.Request) {
    base := BuildDirector(upstream)
    return func(r *http.Request) {
        base(r)
        if authBearer != "" {
            r.Header.Set("Authorization", "Bearer "+authBearer)
        }
        if r.URL.Path != "/v1/chat/completions" || r.Body == nil {
            return
        }
        body, _ := io.ReadAll(r.Body)
        _ = r.Body.Close()
        // 1. rewrite model via resolver
        rewritten, _, _ := models.RewriteJSONModel(body, resolver, upstreamName)
        // 2. inject provider.order + allow_fallbacks
        patched, _ := injectProviderOrder(rewritten, providerOrder, allowFallbacks)
        // 3. inject stream_options.include_usage if streaming
        patched = injectStreamOptionsIncludeUsage(patched)
        r.Body = io.NopCloser(bytes.NewReader(patched))
        r.ContentLength = int64(len(patched))
        r.Header.Set("Content-Length", strconv.Itoa(len(patched)))
    }
}
```

**Important:** `models.RewriteJSONModel` already returns the body unchanged when the alias is unknown (per `rewrite.go:38-42`), so calling it on every chat request is safe â€” the resolver miss returns the alias itself.

## Pitfalls

### Pitfall 1 â€” URL convention `/api` vs `/api/v1`

**What goes wrong:** `BuildDirector` at `director.go:44` deliberately preserves `r.URL.Path` (the comment says "pod routes mirror the gateway's /v1/... paths 1:1"). When `UPSTREAM_LLM_OPENROUTER_URL=https://openrouter.ai/api/v1`, the Director concatenates `https://openrouter.ai/api/v1` + `/v1/chat/completions` = `https://openrouter.ai/api/v1/v1/chat/completions` (double `/v1`). 404.

**Why it happens:** OpenRouter's docs document the base as `https://openrouter.ai/api/v1` but the path templates show `/v1/chat/completions`. Pod base URLs in dev use no `/v1` suffix because the pod mounts routes at `/v1/...` directly.

**How to avoid:** Add boot-time fail-fast validation in `config.go Load()`:

```go
for envName, url := range map[string]string{
    "UPSTREAM_LLM_OPENROUTER_URL": cfg.UpstreamOpenRouterChatURL,
    "UPSTREAM_STT_OPENAI_URL":     cfg.UpstreamOpenAIWhisperURL,
    "UPSTREAM_EMBED_OPENAI_URL":   cfg.UpstreamOpenAIEmbedURL,
} {
    if url != "" && strings.HasSuffix(strings.TrimRight(url, "/"), "/v1") {
        return Config{}, fmt.Errorf("%s must not end with /v1 (gateway preserves request path /v1/...); set to base URL without /v1 suffix (got: %q)", envName, url)
    }
}
```

**Warning signs:** Container fails fast at boot with explicit error message naming the offending env var. Operator updates env in `/opt/ai-gateway-dev/.env` + redeploys.

### Pitfall 6 â€” multipart whisper rewriter must preserve binary file part bytes

**What goes wrong:** Naive multipart rewriter that decodes file as text + re-encodes will corrupt the binary WAV/MP3 audio. The rewriter must `io.Copy` the audio file bytes through without inspection â€” only the small `model` text field gets replaced.

**Why it happens:** `mime/multipart` returns `*Part` with `Read()` that gives raw bytes. If the rewriter wraps in a `bufio.Scanner` or treats as string, multi-byte audio frames are mangled.

**How to avoid:** Use `multipart.NewReader(r.Body, boundary).NextPart()` loop. For each part: if FormName() == "model", read the entire value (small), discard, write replacement value to new writer's field. Otherwise: get a new `Writer.CreatePart(part.Header)` and `io.Copy(newPart, oldPart)`. Never use `string(bytes)` on the audio part.

**Warning signs:** Live UAT S2 returns OpenAI 400 with garbled-audio error; or returns 200 but transcription is garbage. Add a regression test that feeds a known WAV stub (already in `gateway/internal/upstreams/testdata/probe.wav`) through the multipart rewriter and asserts the file bytes survive `bytes.Equal(in, out)`.

### Pitfall 7 â€” sqlc regen forgets to update `GetModelAlias` query

**What goes wrong:** `GetModelAlias :one` at `model_aliases.sql:5` is `WHERE alias = $1`. After PK widening, this query returns multiple rows for any alias (one per upstream). sqlc compiles successfully but the `:one` variant uses `QueryRow` which discards extra rows â€” operator gets unpredictable upstream's target.

**How to avoid:** Either (a) remove `GetModelAlias` (only `ListModelAliases` is used in production code per grep), or (b) extend to `WHERE alias = $1 AND upstream_name = $2`. Verify which call sites use it (`grep -rn GetModelAlias gateway/`).

**Warning signs:** sqlc generate succeeds, code compiles, but runtime returns wrong target for queries by alias-only.

### Pattern 1: JSON body rewrap preserving other fields

**What:** Use `map[string]json.RawMessage` so unknown fields survive byte-identical.

**When to use:** Any time the gateway modifies one field in a JSON body that has client-supplied additional fields.

**Example (existing in repo):**

```go
// Source: gateway/internal/proxy/openrouter_director.go:84-98
func injectProviderOrder(body []byte, order []string, allowFallbacks bool) ([]byte, error) {
    var m map[string]json.RawMessage
    if err := json.Unmarshal(body, &m); err != nil {
        return nil, err
    }
    providerBytes, _ := json.Marshal(map[string]any{
        "order":           order,
        "allow_fallbacks": allowFallbacks,
    })
    m["provider"] = providerBytes
    return json.Marshal(m)
}
```

### Pattern 2: Director composition over inheritance

**What:** Each upstream Director wraps `BuildDirector(upstream)` then adds upstream-specific behavior (auth header, body rewrite).

**When to use:** Adding any new upstream that needs different headers / body shape.

**Example:**

```go
// Source: gateway/internal/proxy/openrouter_director.go:36-77
func BuildOpenRouterDirector(...) func(*http.Request) {
    base := BuildDirector(upstream)
    return func(r *http.Request) {
        base(r) // strips client auth, sets host, propagates X-Request-ID
        // ...upstream-specific work
    }
}
```

### Anti-Patterns to Avoid

- **Request-edge alias rewrite for multi-upstream routing:** Currently `models.Handler` at `main.go:1213-1216` rewrites the alias BEFORE the dispatcher picks an upstream. This collapses N possible per-upstream targets into 1 (the role's "primary" target). Bug surface for all tier-1 upstreams. **Move rewrite to per-upstream Director.**
- **Hard-coded upstream model literals in Director:** `BuildOpenAIEmbedDirector` currently hard-codes `text-embedding-3-small` (line 64). Works today but breaks the schema-as-source-of-truth invariant. Refactor to source from resolver.
- **Multipart re-encoding of audio bytes:** Do not decode WAV/MP3 file part as string. Stream bytes through `io.Copy` from `*Part` to new `Writer.CreatePart`.
- **Removing `models.Handler` without replacement on local path:** The local-llm path also needs the model alias resolved (pod expects `model:"qwen"`, not... well, it currently expects `qwen` which matches the alias). Verify the migration's backfill row `('qwen','local-llm','qwen')` exists so `resolver.Resolve("qwen","local-llm")` returns the unchanged alias.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Multipart form parsing/writing | Custom string-replace on raw bytes | Standard `mime/multipart.NewReader` + `Writer` | Boundary handling + Content-Disposition headers + binary part safety are non-trivial |
| JSON body field rewrite | Custom byte-level surgery | `map[string]json.RawMessage` (existing pattern) | Preserves other client fields byte-identical; works with all OpenAI-compatible payloads |

| URL convention validation | Runtime probe + error capture | Boot-time `strings.HasSuffix` fail-fast | Static config bugs deserve static-time detection |

### Example 2: Multipart-aware Whisper rewriter

```go
// Source: NEW helper (Phase 06.9 introduces)
//
// rewriteMultipartModel parses a multipart/form-data body, replaces the
// `model` form field with newModel, and returns the re-encoded body bytes
// plus the new Content-Type header value (boundary changes on re-encode).
// All other parts (audio file, language, prompt, etc.) are copied through
// byte-identical via io.Copy â€” binary audio data is NEVER decoded as text.
func rewriteMultipartModel(body []byte, contentType, newModel string) ([]byte, string, error) {
    _, params, err := mime.ParseMediaType(contentType)
    if err != nil {
        return body, contentType, fmt.Errorf("parse content-type: %w", err)
    }
    boundary, ok := params["boundary"]
    if !ok {
        return body, contentType, errors.New("no multipart boundary")
    }
    reader := multipart.NewReader(bytes.NewReader(body), boundary)
    var out bytes.Buffer
    writer := multipart.NewWriter(&out)
    for {
        part, err := reader.NextPart()
        if errors.Is(err, io.EOF) {
            break
        }
        if err != nil {
            return body, contentType, fmt.Errorf("read part: %w", err)
        }
        newPart, perr := writer.CreatePart(part.Header)
        if perr != nil {
            return body, contentType, fmt.Errorf("create new part: %w", perr)
        }
        if part.FormName() == "model" {
            if _, werr := io.WriteString(newPart, newModel); werr != nil {
                return body, contentType, werr
            }
            io.Copy(io.Discard, part) // drain old value
        } else {
            if _, cerr := io.Copy(newPart, part); cerr != nil {
                return body, contentType, fmt.Errorf("copy part: %w", cerr)
            }
        }
    }
    if cerr := writer.Close(); cerr != nil {
        return body, contentType, cerr
    }
    return out.Bytes(), writer.FormDataContentType(), nil
}
```
