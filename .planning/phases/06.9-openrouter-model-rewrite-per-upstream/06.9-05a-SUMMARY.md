## Decisions Made

- **R8 gate cleared without code change.** Inspecting `freshSchema` confirmed the helper already returns `(*pgxpool.Pool, *redis.Client)`. The plan's BLOCKING entry gate was a defensive guardrail — Plan 04 (`gatewayctl model-alias` CLI) already needed Postgres-pool access from the helper and the signature was extended in a prior phase. We documented the finding here instead of forcing a no-op edit.
- **Body capture via `atomic.Value`, not `sync.Mutex` + slice.** The handler is the single writer per request; assertion goroutines are downstream readers after the response. `atomic.Value`'s load/store semantics are the right primitive for the "set once per request, read after response" pattern; a mutex would be heavier without buying anything.
- **`newSelectiveMock` lives in resilience_helpers_test.go despite Plan 05a NOT consuming it.** Plan 05b is the consumer (R13 historical-bug regression). Placing the helper in the same file as the body-capture extension keeps `resilience_helpers_test.go` touched exactly ONCE for the Wave 3 split — Plan 05b's diff stays focused on the test files themselves.
- **R6 Test C uses fixture-backed resolver via `useFixtureResolver` parameter.** The migrated DB always has the `(whisper, openai-whisper) → whisper-1` row (migration 0026 seed). To exercise the resolver-miss path without mutating shared schema, the test passes `models.NewResolverForTesting(map[[2]string]string{})` (empty fixture) into the harness. The Director's behavior is identical regardless of resolver source — production code path unchanged.
- **WhisperAbortGuard wiring mirrors production exactly.** The harness wraps the tier-1 ReverseProxy in `proxy.WhisperAbortGuard(...)` BEFORE registering in the dispatcher's `Proxies` map — exactly the shape of `cmd/gateway/main.go` `sttRoleProxies["openai-whisper"]` (Plan 03 wire-up). R6 Test B proves the guard fires end-to-end through the full dispatcher chain, not just at unit-test isolation. WARNING-3 is closed for real.
- **`buildWhisperMultipart` + `parseWhisperMultipart` reimplemented in the test file.** The proxy package already has equivalent helpers (`buildMultipartBody` / `parseMultipartFromBytes` in `openai_whisper_director_test.go`) but those live in `package proxy` and the integration_test package can't import them. The new helpers are byte-identical in semantics (verified by inspection); duplicating ~30 lines of test code is cheaper than refactoring helper packaging.
- **`http.NoBody` not used.** Initial draft of the embed test tried to use `http.NoBody` + manual body replacement to avoid a stale draft. Final version uses `httptest.NewRequest(POST, path, strings.NewReader(bodyJSON))` directly, which is idiomatic. (Fixed mid-implementation; never committed.)
- **Comment language for duplicate-model test avoids the literal "t.Skip" string.** The plan's verification `! grep -q "t.Skip" openai_whisper_model_rewrite_test.go` would otherwise false-positive on a comment that mentioned `t.Skip`. Comment reworded to "MUST PASS (never skipped) per Plan 03 WARNING-3 wire-up". No semantic change.
